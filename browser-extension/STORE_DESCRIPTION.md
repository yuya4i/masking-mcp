# Chrome Web Store 詳細説明 (~v1.3.0)

以下をそのまま「詳細な説明 (Detailed description)」欄にコピペ可能。

---

## なぜ PII Guard が必要なのか

ChatGPT、Claude、Gemini、Manus などの生成 AI サービスにアイデアや業務文書を貼り付けるとき、その中に紛れ込んでいる電話番号・メールアドレス・顧客名・社員 ID・API キー・銀行口座番号 — それらは AI 提供者のサーバーに**そのまま記録され**、モデル学習やログ保存の対象になる可能性があります。

**PII Guard** はあなたが Enter を押す**直前**にブラウザ側でテキストを走査し、個人情報・機密情報を検出してプレースホルダー (`<EMAIL_1>` `<PHONE_1>` など) に置換した状態で送信します。すべての処理はあなたのブラウザ内で完結し、拡張機能の開発者が運営するサーバーへ通信することは一切ありません。

---

## 3 系統の検出バックエンド

### 1. Regex + 多言語辞書 (デフォルト ON、常時オフライン)

40+ カテゴリのプリセット正規表現と、47 都道府県・政令指定都市 20・主要国名 30+・苗字 50・主要 API キーベンダー 30+ を内蔵した辞書による検出。インストール直後から動作し、外部通信は一切発生しません。

### 2. ローカル LLM 連携 (オプトイン、v1.2.0+)

Ollama / LM Studio / llama.cpp など OpenAI 互換 API を指定すると、文脈考慮型の PII 検出 + AI 置換モードが利用可能。`http://localhost:11434` などユーザー指定の LAN 上 LLM サーバーにのみ通信し、Service Worker の host-lock で他ホストへの転用は物理的にブロックされます。Chrome Web Store 版では `optional_host_permissions` を採用しており、ユーザーが明示的に有効化した時に限り Chrome がランタイム権限プロンプトを表示します。

### 3. ブラウザ内 ML 検出 (オプトイン、v1.3.0+ NEW)

DistilBERT-multilingual NER (約 135MB int8 量子化) を **Hugging Face 公式ホストから 1 回だけ取得** しブラウザ内 IndexedDB にキャッシュ。以降はオフラインで人名・地名・組織名を文脈から検出します。LLM サーバー不要で、Docker や追加インフラなしでセマンティック検出が動作します。モデル取得時に **ユーザー入力テキストや個人情報は一切送信されません** — モデル ID とファイル名のみが伝わります。

---

## 40+ カテゴリの自動検出

正規表現とローカル NLP ロジック、**静的辞書 fallback 層**、それに加えて v1.3.0 から **ブラウザ内 ML 検出** を組み合わせ、以下を自動検出します:

- **連絡先**: 電話番号 (固定/携帯/0120 等)、メールアドレス、住所、郵便番号
- **地名**: 都道府県+市区町村 (例: 兵庫県明石市)、47 都道府県単体、政令指定都市 20 単体、主要国名 (日英両表記)、ML 検出による文脈考慮型地名 (v1.3.0+)
- **人名**: 日本人苗字 top 50 辞書 (佐藤/鈴木/高橋/田中 等、多字のみ)、Western first names (John/Mary/David 等)、ML 検出による多言語人名 NER (v1.3.0+)
- **個人識別**: マイナンバー、運転免許証、パスポート、患者 ID
- **金融**: クレジットカード番号、銀行口座 (普通/当座)、年収、月収
- **企業**: 株式会社/㈱/有限会社の社名、組織名、社員 ID、契約番号、発注番号、請求書番号、顧客 ID、会員 ID、SKU、ML 検出による組織名 NER (v1.3.0+)
- **クレデンシャル**: API キー (30+ ベンダー固有プレフィックス: OpenAI / Anthropic / GitHub / AWS / Slack / Google / Stripe / Perplexity / OpenRouter 等)、シークレット、DB 接続文字列、IP アドレス、URL、JWT、PEM 鍵
- **その他**: 年齢、性別、生年月日、血液型、特許番号、資産番号、ライセンス番号

## 重要度別の対応

検出された情報は **Critical / High / Medium / Low** の 4 段階に自動分類されます:

- **Critical** (赤): マイナンバー、API キー、銀行口座、人名、㈱付き会社名、ドメイン付メールなど — 漏洩すると即時被害が出る情報
- **High** (橙): 電話番号、住所、患者 ID
- **Medium** (黄): 部署名、契約番号、金額、URL
- **Low** (灰): 年齢、性別、日付、郵便番号

サイドバーには重要度別タブ (All / Critical / High / Medium / Low) があり、確認したい階層だけ即座にフィルタできます。

## 送信前のレビュー画面

「インタラクティブモード」(初期 ON) では、Enter を押した瞬間にサイドバーが開き、**マスク対象の一覧 + プレビュー** が表示されます。

- 各検出を **クリック 1 回でマスク ON/OFF** 切替
- マスク OFF にした行は右側に**緑バー**が表示され「これは原文のまま送信される」ことを視覚的に確認可能
- マスク ON 時は元の値が薄く非アクティブに、プレースホルダーが強調表示
- マスク OFF 時は逆転 — どちらが実際に送信されるかが一目で分かるデザイン
- 「すべて選択 / すべて解除」ボタンで一括操作も可能
- リアルタイムプレビューで、最終的に AI に届くテキストの全体像が見える

### 機密キーワードによる自動ロック

「機密」「リーク」「未公開」「confidential」「leak」のいずれかが本文に含まれる場合、**人名・組織名・財務情報のカテゴリは自動的にロック**されます。ロック行は 🔒 アイコン付きで表示され、長押しゲート (0〜1.5 秒、設定で調整可) を経由しないと解除できません。

### ドラッグ & ドロップでマスク対象に追加

拡張機能の検出ルールに引っかからない社内用語やプロジェクト名も、ワンアクションでマスク対象に登録できます。

- チャット画面で文字列を選択 → **サイドバーへドラッグ** → カテゴリを 9 種 (PERSON / LOCATION / ORGANIZATION / CONTACT / FINANCIAL / CREDENTIAL / IDENTITY / INTERNAL_ID / OTHER) から選択
- 選択したカテゴリに応じて severity (critical / high / medium / low) が自動付与
- 登録された値は以降すべての入力で必ずマスクされ、全サイトで共有
- 既に検出済みの値をドラッグした場合は **該当行へ自動スクロール + 2 秒黄色フラッシュ**
- サイドバー行の「✖ 削除」ボタンで force-list からワンクリック除去
- **リアルタイム再集約**: 「田中」検出中に「田中 太郎」を追加すると、重なる箇所は「田中 太郎」が勝ち、交差しない「田中」は件数を自動で減らして残る

### マスキング除外リスト (個人専用ホワイトリスト)

社名・プロジェクト名・自分の名前など「PII として検出されるが自分にとってはマスキング不要」な値を **永続登録**できます。

- 拡張機能の設定画面 (オプションページ) でフルスペックの CRUD 管理
- サイドバーの各行にある「✖ 除外」ボタンから 1 クリック登録
- **JSON エクスポート/インポート対応** — 別 PC への移行やチーム内共有が可能
- 別タブで除外リストに追加すると、現在開いているサイドバーに**即時反映** (リロード不要)

## モダンな UI

- ホストページの背景色を自動検出し、ライト/ダークテーマを切り替え (ChatGPT ダークモードなどでも違和感なし)
- サイドバーは固定オーバーレイではなく **push 型** — チャット領域を左に押し出すので会話が隠れません
- ビューポートサイズに応じて 400px → 45vw まで縮小
- カテゴリはデフォルト折りたたみ表示。「▾ すべて展開 / ▸ すべて折りたたむ」一括トグル
- 該当行ジャンプ時は親カテゴリを自動展開

### Socket.IO (WebSocket) 対応

manus.im など WebSocket でメッセージを送るチャット UI にも対応。Engine.IO / Socket.IO のフレーム構造を理解し、ユーザー入力部分のみを抽出してマスキング。ping/pong や認証フレームには触らないので、チャット session が切断されることはありません。

## プライバシー & セキュリティ

**ローカル中心動作です。**

- すべての PII 検出ロジックは拡張機能内にバンドルされた純粋な JavaScript で実装
- 入力テキスト・検出結果・マスク済みテキストが**拡張機能の開発者・Anthropic・OpenAI・Hugging Face・その他いかなる第三者クラウド**へ送信されることはありません
- 拡張機能が動作するのは **claude.ai / chatgpt.com / openai.com / gemini.google.com / manus.im** の 6 ホストのみ。他のサイトでは一切起動しません
- 動的コード生成 (eval 系 API) は一切使用しない静的バンドル

**v1.2.0+ オプション通信:** ローカル LLM 連携を有効化した場合のみ、ユーザーが指定したローカル/LAN 上の LLM サーバー (Ollama 等) に PII 検出クエリが送信されます。Service Worker の host-lock で他ホストへの転用は物理的に不可能です。

**v1.3.0+ オプション通信:** ブラウザ内 ML 検出を有効化した場合のみ、Hugging Face 公式ホストから **1 回限り** モデルファイル (~135MB) が取得されます。モデル取得時にユーザー入力テキストは送信されません。取得後の推論は完全オフラインです。

**ブラウザ内ストレージに保存される情報:**

- マスキング機能の ON/OFF
- インタラクティブモードの ON/OFF
- UI モード (サイドバー / モーダル)
- 除外リスト・強制マスクリストに登録した文字列
- ローカル LLM 連携設定 (URL / モデル名 / タイムアウト)
- ブラウザ内 ML 検出の有効化フラグ (v1.3.0+)
- タブごとの検出件数 (バッジ表示用、タブを閉じると消去)

これらはあなたの端末内 Chrome ストレージにのみ保存され、開発者を含む第三者がアクセスすることはありません。拡張機能をアンインストールすると同時に全削除されます。

詳細はプライバシーポリシーをご確認ください: https://github.com/yuya4i/pii-masking/blob/main/browser-extension/PRIVACY.md

## 対応サービス

| サービス | URL | 検出経路 |
|---|---|---|
| Claude | claude.ai / *.claude.com | fetch / XHR |
| ChatGPT | chatgpt.com / *.openai.com | fetch / XHR |
| Gemini | gemini.google.com | fetch / XHR |
| Manus | *.manus.im | WebSocket (Socket.IO) |

それぞれのサービスごとに、メッセージ送信時の payload フォーマットを理解した「アダプタ」が設計されており、誤検出やアプリ側機能の破壊なしに動作します。

## こんな方におすすめ

- 業務で AI を使うが、社内情報・顧客情報の取り扱いガイドラインに従う必要がある方
- 個人 PC で複数プロジェクトを抱え、うっかり別案件の名前を貼り付けてしまう不安がある方
- API キーやアクセストークンをコピペでうっかり共有してしまった経験がある方
- AI サービス側のログ保持/学習ポリシーを完全には信頼しきれないと感じる方
- ローカル PC のみで完結するソリューションを求める方
- ローカル LLM (Ollama) を使った文脈考慮型マスキングを試したい開発者

---

## 使い方

1. 拡張機能をインストールし、対応する AI サービスを開く (権限要求は最小限 — chat 6 ホストのみ)
2. いつも通りメッセージを入力して送信ボタンを押す
3. 個人情報が検出されると自動的にレビューサイドバーが開く
4. マスク対象を確認し、必要に応じて個別に ON/OFF
5. 「マスクして送信」ボタンで確定 (または Esc でキャンセル)

「サイドバーをスキップして自動マスクで送信したい」場合は、ポップアップから「送信前に確認する」を OFF にしてください。

オプション画面 (`chrome://extensions` → 詳細 → オプション) で:

- マスキング除外リストの管理
- ローカル LLM 連携の有効化 (Ollama 等の URL を指定)
- ブラウザ内 ML 検出の有効化 (Hugging Face Hub からモデル DL、1 回限り、v1.3.0+)

## 既知の制限

- **日本語人名**: top 50 の苗字 (佐藤/鈴木/高橋/田中 等、多字のみ) を辞書収録 + v1.3.0 から ML 検出が文脈考慮で人名検出を補強。単字の姓 (林/森/川 等) は一般名詞との衝突を避けて辞書から除外
- **英語の固有名詞**: business-common な first name 26 種を辞書収録 + v1.3.0 ML 検出で姓・組織名の NER に対応
- **画像内のテキスト**: OCR は未対応。テキスト入力欄のみが対象
- **ストリーミング応答**: AI からの応答中に含まれる PII は対象外 (送信時のみ)
- **ローカル LLM のコールドスタート**: 初回 20-90 秒 (4B+ モデルの場合)
- **ML 検出のモデル取得**: 初回有効化時 ~135MB ダウンロード (約 30-90 秒、以降キャッシュ)
